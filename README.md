## 📌 Project Description

A fully semantic, accessible, and professionally designed **static portfolio webpage** built from scratch using only **HTML5 and CSS3**, no frameworks, no libraries, no shortcuts.

This is **Project 1** of the DecodeLabs Frontend Development internship track (Batch 2026). The goal is to prove mastery of the fundamental relationship between content and style before touching any dynamic logic.

## ✨ Features

- **Semantic HTML5** : proper use of `<header>`, `<nav>`, `<main>`, `<section>`, `<article>`, `<aside>`, `<footer>`
- **Single `<h1>`** per page with correct heading hierarchy (h1 → h2 → h3)
- **CSS Grid** for macro page layout and section structure
- **Flexbox** for micro component alignment
- **BEM Methodology** : structured, reusable class naming
- **DRY Principle**: no repeated styles, external CSS only, zero inline styles
- **Accessibility (A11y)** : alt text, aria-labels, WCAG 4.5:1 contrast ratio
- **Responsive Design** : mobile-friendly with media query breakpoints
- **Natural Light Theme**: warm parchment tones, forest green accents, terracotta highlights
- **Smooth animations**: subtle float, fade-up, and hover transitions via CSS only

## 🚀 How to Run

This is a pure static project - no installation, no build tools, no terminal commands needed.

**Option 1 - Open directly in browser**
```
1. Download or clone this repository
2. Open the project folder
3. Double-click index.html
4. It opens in your default browser instantly
```

**Option 2 - Clone via Git**
```bash
git clone https://github.com/YOUR-USERNAME/decodelabs-frontend-p1.git
cd decodelabs-frontend-p1
open index.html
```

**Option 3 - Live Server (recommended for development)**
```
1. Open the folder in VS Code
2. Install the "Live Server" extension
3. Right-click index.html → "Open with Live Server"
4. Auto-refreshes on every save
```

## 📁 Commit History Approach

This project follows a **professional commit discipline** - every section was committed separately in structure-before-style order, mirroring the IPO model taught in the DecodeLabs brief.

```
feat(css):  add footer styles
feat(html): add footer structure
feat(css):  add contact section styles
feat(html): add contact section structure
feat(css):  add philosophy section styles
feat(html): add philosophy section structure
feat(css):  add projects section styles
feat(html): add projects section structure
feat(css):  add skills section styles
feat(html): add skills section structure
fix(html):  fix structural and typo bugs
feat:       add personal photo to about section
feat(css):  add hero section styles
feat(html): add hero section structure
feat(css):  add CSS foundation — tokens, reset, utilities, header
Initial commit
```
